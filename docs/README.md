# Docs

Indice de documentacion del proyecto.

## Documentos de entrada

- `project-status.md`: estado real del proyecto, riesgos y nivel de madurez actual.
- `improvements-and-next-steps.md`: backlog priorizado de mejoras y plan recomendado.

## Documentos tecnicos

- `architecture.md`: organizacion del firmware y responsabilidades por modulo.
- `hardware-notes.md`: resumen del hardware confirmado y puntos aun por validar.

## Documentos de diseno

- `visual-spec.md`: direccion visual, personalidad y reglas de estados de EMOpet.
- `design-system.md`: tokens, componentes, jerarquia visual y escala base.
- `ui-screens.md`: layouts de vistas, rotacion y comportamiento tactil.

## Recursos visuales

- `renders/`: imagenes de apoyo para vistas y composicion.
